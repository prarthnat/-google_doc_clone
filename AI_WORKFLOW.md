## 2. Where AI Materially Sped Up Work

### A. Boilerplate & UI Component Scaffolding (60% Time Saved)
Building a visually polished Google Docs clone with Tailwind CSS requires significant markup for rich formatting toolbars, account switchers, modals, and responsive grids. By instructing the AI with precise, high-signal prompts specifying exact design tokens (`brand-500`, `shadow-paper`, glassmorphism, crisp Lucide icons), the initial frontend scaffolding across `Navbar.tsx`, `Sidebar.tsx`, and `DocumentList.tsx` was generated in minutes rather than hours.

### B. Relational Schema & Seeding Logic (50% Time Saved)
AI accelerated writing the `better-sqlite3` table initialization and seeding script (`server/db.ts`). By defining our exact requirements (4 demo users, `document_shares` join table with `permission` constraints, and rich sample documents with HTML formatting), the AI generated valid, high-performance SQL with `WAL` mode and foreign key pragmas on the first pass.

### C. Automated Test Authoring (`Vitest + Supertest`)
Instead of manually typing out tedious HTTP request payloads, I instructed the AI to generate our `server/tests/documents.test.ts` suite to verify `201 Created`, `403 Forbidden` access blocks, and markdown regex parsing. This allowed us to run `npm test` continuously and catch regressions instantly.

## 3. What AI-Generated Output Was Changed or Rejected

As an **AI-Native Full Stack Product Engineer**, I treat AI outputs as high-speed rough drafts that require strict human judgment, refactoring, and quality assurance:

1. **Rejected Over-Engineered WebSocket CRDT Suggestions:**
   - *AI Suggestion:* Early prompts suggested integrating `Yjs` with `y-webrtc` or an external `Hocuspocus` WebSocket server for live multiplayer sync.
   - *Human Judgment:* I rejected this because it would force hiring managers reviewing the project to run external WebSocket relays or debug port conflicts. Instead, I designed a clean **debounced REST + SQLite autosave architecture** with an **instant Account Switcher**, ensuring 100% self-contained reliability.

2. **Fixed Hallucinated Icon Exports (`lucide-react`):**
   - *AI Error:* During build compilation (`npm run build`), the AI generated an import for `CloudCheck` from `lucide-react`, which caused a Rollup bundling failure because that icon does not exist in the exact package version.
   - *Human Correction:* I diagnosed the trace, identified the correct icon token (`CheckCircle`), and executed targeted replacements to ensure clean, zero-error production builds.

3. **Refined Debounced Autosaving & Race Conditions:**
   - *AI Suggestion:* The raw TipTap `onUpdate` callback generated by AI fired `fetch` requests on every single keystroke.
   - *Human Refinement:* I wrapped the API calls in a `useRef(setTimeout)` debounce (`700ms`) and added explicit `Saving...` / `Saved to SQLite` visual feedback states to prevent network flood and SQLite file-lock collisions.

## 4. How Verification & UX Quality Were Maintained

### Engineering Quality Checks
- **Automated Verification:** Ran `npm test` after every major backend route addition to guarantee zero breaking changes across permissions (`owner`, `edit`, `view`).
- **Production Bundle Validation:** Executed `npm run build` to verify that TypeScript compilation (`tsc --project tsconfig.server.json`) and Vite static asset bundling succeed without type mismatches or unhandled promises.

### UX Quality & Polish
- **Physical Paper Aesthetic:** Inspected the editor container across desktop and tablet viewports to ensure the paper card (`max-w-4xl min-h-[900px] shadow-paper`) looks identical to real document authoring software.
- **Graceful Error Handling:** Verified that when a `View Only` user tries to type or edit title input, the UI gracefully blocks editing without throwing console exceptions while displaying a helpful warning banner.
